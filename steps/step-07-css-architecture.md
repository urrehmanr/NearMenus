# Step 7: CSS Architecture & Global Styles

## Objective
Create a maintainable, performant CSS architecture that works seamlessly with theme.json, provides consistent styling across all templates, and optimizes for Core Web Vitals performance metrics.

## What You'll Learn
- Modern CSS architecture principles
- Integration with theme.json variables
- Performance-optimized CSS structure
- Responsive design patterns
- CSS custom properties usage
- Critical CSS strategies

## CSS Architecture Overview

Our CSS architecture follows these principles:
1. **Mobile-first responsive design**
2. **CSS custom properties from theme.json**
3. **Minimal cascade and specificity**
4. **Component-based organization**
5. **Performance-optimized loading**
6. **Utility-first patterns where appropriate**

## File Structure

```
assets/css/
├── style.css              # Main development CSS
├── style.min.css          # Minified production CSS
├── editor-style.css       # Block editor styles
├── editor-style.min.css   # Minified editor styles
├── responsive.css         # Responsive breakpoints
├── responsive.min.css     # Minified responsive
├── print.css              # Print-specific styles
├── print.min.css          # Minified print styles
└── critical.css           # Critical above-the-fold CSS
```

## Build Process Integration

### Using PostCSS for optimization:

Create `postcss.config.js`:
```javascript
module.exports = {
  plugins: [
    require('autoprefixer'),
    require('cssnano')({
      preset: 'default',
    }),
  ],
}
```

### Build scripts in package.json:
```json
{
  "scripts": {
    "build:css": "postcss assets/css/style.css -o assets/css/style.min.css",
    "build:css-responsive": "postcss assets/css/responsive.css -o assets/css/responsive.min.css",
    "build:css-editor": "postcss assets/css/editor-style.css -o assets/css/editor-style.min.css",
    "build:css-critical": "postcss assets/css/critical.css -o assets/css/critical.min.css",
    "build:all-css": "npm run build:css && npm run build:css-responsive && npm run build:css-editor && npm run build:css-critical",
    "watch:css": "postcss assets/css/style.css -o assets/css/style.min.css --watch"
  }
}
```

## Performance Optimization Strategies

### 1. Critical CSS Inlining
```php
// In functions.php
function modernblog2025_inline_critical_css() {
    if (is_front_page() || is_home()) {
        $critical_css = file_get_contents(MODERNBLOG2025_THEME_DIR . '/assets/css/critical.min.css');
        echo '<style id="critical-css">' . $critical_css . '</style>';
    }
}
add_action('wp_head', 'modernblog2025_inline_critical_css', 1);
```

### 2. CSS Purging
Remove unused CSS in production:
```javascript
// purgecss.config.js
module.exports = {
  content: [
    './templates/**/*.html',
    './parts/**/*.html',
    './*.php',
    './inc/**/*.php'
  ],
  css: ['./assets/css/style.css'],
  safelist: [
    /^wp-/,
    /^has-/,
    /^is-/
  ]
}
```

## Troubleshooting

**Styles not applying:**
- Check CSS syntax validity
- Verify file paths in enqueue functions
- Clear browser and WordPress cache
- Check theme.json custom property names

**Performance issues:**
- Minimize CSS file sizes
- Use critical CSS for above-the-fold content
- Optimize CSS delivery
- Remove unused styles
